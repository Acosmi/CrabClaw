# 微信公众号全流程运营

媒体运营子智能体 — 微信公众号平台专属技能。覆盖图文创作规范、API 发布流程、素材管理。

---

## 平台硬限制

| 项 | 限制 | 说明 |
|---|---|---|
| 标题 | **≤64 字** | 中文字符算 1 个 |
| 正文 | 无硬性限制 | 建议 1500~3000 字 |
| 正文格式 | **HTML** | API 接口要求 HTML |
| 封面图 | **必须** | 通过 API 上传获得 thumb_media_id |
| 图片格式 | **JPG / PNG** | 不支持 GIF / WebP |
| 图片大小 | **≤1 MB** / 张 | 超出则上传失败 |
| 发布模式 | **异步** | 提交后轮询状态 |
| Token 有效期 | **2 小时** | 5 分钟提前刷新 |

## 内容类型

微信公众号以长图文为核心，适合深度内容。

### 按内容方向分

| 方向 | 特点 | 标题套路 | 风格 |
|------|------|----------|------|
| **深度分析** | 数据 + 观点 + 论证 | "X背后的真相/启示" | professional |
| **行业资讯** | 时效 + 信息密度 | "本周/今日X大看点" | informative |
| **教程指南** | 步骤 + 截图 + 实操 | "从零开始/完全指南" | informative |
| **观点评论** | 立场鲜明 + 论据充分 | "为什么说…/我们该如何…" | professional |
| **案例故事** | 叙事 + 分析 + 启发 | "从X案例看…" | casual/professional |
| **盘点合集** | 结构化 + 可收藏 | "年度/必看/最全盘点" | informative |

## 创作规范

### 标题写法

- **控制在 64 字以内**，理想 15~30 字
- 开头放核心信息（微信列表只显示前 20 字左右）
- 使用分隔符增加节奏：`|` `：` `——`
- 数字具体化：`3个方法` `7天见效` `80%的人不知道`

**好标题**：
- `深度解析：2026年AI行业三大趋势与应对策略`
- `从0到1搭建个人知识库｜附完整工具清单`
- `为什么你的简历总是石沉大海？HR不会告诉你的5个真相`

**坏标题**：
- `好文推荐`（无信息量）
- `震惊！！！必看！！！`（标题党）

### 正文结构

微信图文适合长篇深度内容，推荐以下结构：

```
[大标题]

[导语 - 100字以内，点明主题和阅读价值]

## 一、[第一部分标题]

[论述段落 - 3~5段，每段3~5行]
[配图/数据/引用]

## 二、[第二部分标题]

[论述段落]
[配图/数据/引用]

## 三、[第三部分标题]

[论述段落]

## 总结

[核心观点复述 + 行动号召]
[互动引导：欢迎留言讨论/转发给需要的朋友]
```

### 正文写法

- 建议 **1500~3000 字**，太短缺深度，太长掉阅读
- 小标题划分板块，读者可跳读
- 每段 3~5 行，避免大段文字墙
- 关键结论 **加粗** 突出
- 适当使用引用块、列表、表格增强可读性
- 数据和观点注明来源

### 正文 HTML 格式

微信 API 要求 HTML 格式正文。系统会自动将纯文本换行转为 `<p>` 标签。

常用标签：
- `<p>段落文本</p>` — 段落
- `<strong>加粗</strong>` — 强调
- `<em>斜体</em>` — 辅助强调
- `<blockquote>引用</blockquote>` — 引用块
- `<ul><li>列表项</li></ul>` — 无序列表
- `<h2>小标题</h2>` — 段落标题
- `<img src="..."/>` — 配图（需先上传获取 URL）

### 封面图

- **必须设置**，否则发布失败
- 建议比例 **2.35:1**（900x383 像素）
- 第一张图片自动用作封面图（`thumb_media_id`）
- 格式 JPG/PNG，大小 ≤1MB
- 封面要有信息量，避免纯色/纯文字

## 发布流程（API 模式）

微信公众号通过官方 API 发布，非 RPA。

### 前置条件

1. AppID + AppSecret 已配置
2. access_token 可正常获取（2 小时有效，自动刷新）
3. 草稿状态为 `approved`

### 发布步骤

```
content_compose { action: "draft", platform: "wechat", ... }
  ↓ 获得 draft_id
回报主智能体等待审批
  ↓ 审批通过
media_publish { action: "status", draft_id: "<id>" }
  ↓ 确认 status == "approved"
media_publish { action: "publish", draft_id: "<id>", platform: "wechat" }
  ↓ 返回 publish_id
media_publish { action: "status", publish_id: "<id>" }
  ↓ 轮询直到 status == "published" 或 "failed"
```

### API 内部流程

1. **上传封面图** — `UploadImage()` → 获得 `media_id`
   - 校验：仅 .jpg/.jpeg/.png，≤1MB
   - 失败则跳过（会使用默认封面）
2. **创建草稿** — `CreateDraft()` → 获得 `draft_media_id`
   - 正文自动包裹 `<p>` 标签
3. **提交发布** — `SubmitPublish(draft_media_id)` → 获得 `publish_id`
4. **查询状态** — `GetPublishStatus(publish_id)` → 三种结果
   - `status=0` → `published`（成功，返回 article_id + article_url）
   - `status=1` → `publishing`（进行中，需继续轮询）
   - `status≥2` → `failed`（失败，查看 fail_idx）

### 发布频率

- 订阅号：**每天 1 次**群发
- 服务号：**每月 4 次**群发
- API 请求频率：≥50ms 间隔（内置限流）

### access_token 机制

- 有效期 **7200 秒**（2 小时）
- 系统自动提前 **5 分钟**刷新
- 多账号独立管理，互不干扰
- Token 可持久化缓存（`token_cache_path` 配置）

### 错误处理

| 微信 errcode | 含义 | 处理 |
|---|---|---|
| 40001 | access_token 无效 | 自动重获取 |
| 40004 | 不合法的媒体文件类型 | 检查图片格式 |
| 45008 | 图文消息超过限制 | 检查正文内容 |
| 45028 | 每日群发次数超限 | 等待次日 |
| 48001 | API 未授权 | 检查 AppID 权限 |

## 素材管理

### 图片素材

上传流程：
1. 验证文件格式（.jpg / .jpeg / .png）
2. 验证文件大小（≤1MB）
3. 调用 `cgi-bin/material/add_material` API
4. 返回 `media_id`（永久素材）

注意：
- 永久素材有**数量上限**（图片 5000 个）
- 临时素材 3 天过期，不推荐使用
- 上传失败不阻塞发布，但封面图会缺失

### 正文配图

正文中的图片需先上传到微信服务器：
1. 上传图片 → 获得 URL
2. 在 HTML 正文中用 `<img src="URL"/>` 引用
3. 不能直接引用外部图片链接（微信会过滤）

## 内容审查清单

草稿提交前逐项检查：

- [ ] 标题 ≤64 字，核心信息在前 20 字
- [ ] 正文 1500~3000 字（建议范围）
- [ ] 正文为 HTML 格式或纯文本（系统自动转换）
- [ ] 至少 1 张封面图，JPG/PNG，≤1MB
- [ ] 文章结构清晰，有小标题分段
- [ ] 数据和引用有来源标注
- [ ] 无违规内容（政治敏感、虚假信息、侵权）
- [ ] 有互动引导（留言/转发/在看）
- [ ] 内容原创，非洗稿搬运

## 与小红书的核心差异

| 维度 | 微信公众号 | 小红书 |
|------|-----------|--------|
| 内容长度 | 长篇深度（1500~3000字） | 短平快（≤1000字） |
| 格式 | HTML，结构化排版 | 纯文本 + emoji |
| 图片角色 | 辅助说明 | 核心主角 |
| 发布方式 | API（异步） | RPA（浏览器自动化） |
| 互动形式 | 精选留言 | 评论 + 私信 |
| 语言风格 | 书面/专业 | 口语/亲切 |
| 分发逻辑 | 推送给已关注用户 | 推荐算法分发 |
| 认证要求 | AppID + AppSecret | Cookie 登录 |
| 标签 | 不强制 | 3~5个必须 |
