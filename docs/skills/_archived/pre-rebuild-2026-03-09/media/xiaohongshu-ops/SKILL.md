# 小红书全流程运营

媒体运营子智能体 — 小红书平台专属技能。覆盖内容创作规范、RPA 发布流程、社交互动策略。

---

## 平台硬限制

| 项 | 限制 | 违反后果 |
|---|---|---|
| 标题 | **≤20 字** | 截断或发布失败 |
| 正文 | **≤1000 字** | 超出部分不展示 |
| 图片 | 每篇 **≤9 张** | 多余图片被忽略 |
| 标签 | `#话题#` 格式，建议 **3~5 个** | 过多标签影响推荐 |
| 操作间隔 | **≥5 秒** + 随机延迟 | 触发风控 |
| Cookie 有效期 | 需定期刷新 | 过期则 RPA 失败 |

## 内容类型

小红书内容以笔记为核心，根据目标选择不同类型：

### 图文笔记（主力）

适合种草、教程、测评、经验分享。

**结构模板**：
```
[标题 - 数字/疑问/痛点开头，≤20字]

[首句 - 制造共鸣或好奇]

[正文分点 - 每点2~3句]
1️⃣ 要点一
2️⃣ 要点二
3️⃣ 要点三

[结尾 - 互动引导]
你们觉得呢？评论区聊聊~

#标签1# #标签2# #标签3#
```

### 按内容方向分

| 方向 | 特点 | 标题套路 | 风格 |
|------|------|----------|------|
| **种草推荐** | 真实体验 + 使用感受 | "用了X天，真的绝了" | casual |
| **干货教程** | 步骤清晰 + 可操作 | "手把手教你/X个方法" | informative |
| **测评对比** | 客观数据 + 主观感受 | "A vs B 真实对比" | professional |
| **经验分享** | 亲身经历 + 避坑 | "踩了X个坑后的总结" | casual |
| **合集清单** | 多项推荐 + 分类 | "X个值得收藏的…" | informative |

## 创作规范

### 标题写法

- **控制在 20 字以内**，每个字都要有信息量
- 开头用数字：`5个` `3步` `10分钟`
- 制造好奇：`居然` `没想到` `后悔没早知道`
- 痛点共鸣：`别再…了` `千万别踩坑`
- 避免标题党和虚假承诺

**好标题**：
- `3步搞定早餐，上班族必看`（17字）
- `用了半年，这5样真的回购了`（13字）
- `新手别踩坑！装修前必看清单`（13字）

**坏标题**：
- `这个东西真的太好用了你一定要试试不然会后悔的`（超20字）
- `震惊！！！绝了！！！`（无信息量）

### 正文写法

- **总字数 ≤1000 字**，理想 300~800 字
- 分段清晰，每段不超过 3~4 行
- 适度使用 emoji（每段 1~2 个，不堆砌）
- 口语化表达，像跟朋友聊天
- 避免长句和书面语
- 关键信息用 **加粗** 或 emoji 标注

### 标签策略

- 固定 3~5 个标签
- 结构：1 个大热标签 + 2~3 个精准标签 + 1 个长尾标签
- 示例：`#美食推荐# #早餐食谱# #上班族便当# #10分钟料理#`
- 标签紧跟正文末尾，用 `#话题#` 格式

### 图片要求

- 首图决定点击率 — 信息量大、色彩鲜明
- 封面图建议 3:4 竖版比例
- 每张图片聚焦一个信息点
- 文字标注简洁（≤10字/张）
- 真实场景 > 过度修图

## 发布流程（RPA 模式）

小红书通过浏览器 RPA 发布，基于 Cookie 登录。

### 前置条件

1. Cookie 文件已加载（`LoadCookies()`）且未过期（`CheckCookieValid()`）
2. BrowserDriver 已注入（`SetBrowser()`）
3. 草稿状态为 `approved`

### 发布步骤

```
content_compose { action: "draft", platform: "xiaohongshu", ... }
  ↓ 获得 draft_id
回报主智能体等待审批
  ↓ 审批通过
media_publish { action: "status", draft_id: "<id>" }
  ↓ 确认 status == "approved"
media_publish { action: "publish", draft_id: "<id>", platform: "xiaohongshu" }
```

### RPA 内部流程

1. 导航到 `creator.xiaohongshu.com/publish/publish`
2. 注入 Cookie → 刷新页面
3. 等待编辑区加载（`.ql-editor`）
4. 上传图片（`input[type='file']`）
5. 填写标题（`#title` / `[placeholder*='标题']`）
6. 填写正文（`.ql-editor` / `[contenteditable]`）
7. 逐个输入标签（`#话题` 触发联想 → 点选）
8. 点击发布按钮（`.publish-btn`）
9. 等待发布成功（`.publish-success`）

### 频率限制

- 每次操作间隔 **≥5 秒**（配置 `rate_limit_seconds`，最低 3 秒）
- 额外 **0~2 秒**随机延迟，模拟人工
- 连续发布建议间隔 **30 分钟**以上

### 错误处理

| 错误 | 原因 | 处理 |
|---|---|---|
| `browser driver not configured` | 未注入 BrowserDriver | 检查 Gateway 配置 |
| `no cookies loaded` | Cookie 文件未加载 | 调用 LoadCookies |
| `cookie expired` | Cookie 过期 | 需重新登录获取 |
| 发布确认超时 | DOM 结构变化 | 查看错误截图 `_media/xhs/errors/` |

## 互动运营

### 评论管理

```
# 查看笔记评论
social_interact { action: "list_comments", note_id: "<id>" }

# 回复评论
social_interact {
  action: "reply_comment",
  note_id: "<id>",
  comment_id: "<id>",
  reply: "回复内容"
}
```

### 私信管理

```
# 查看私信
social_interact { action: "list_dms" }

# 回复私信
social_interact {
  action: "reply_dm",
  user_id: "<id>",
  message: "回复内容"
}
```

### 回复话术

| 评论类型 | 策略 | 示例 |
|---|---|---|
| 求链接/求推荐 | 正面回应 + 引导关注 | "宝子可以看看我主页其他笔记哦~" |
| 使用提问 | 直接解答 + 补充 | "可以的呢，我当时是这样做的…" |
| 正面反馈 | 真诚感谢 + 延伸 | "谢谢喜欢！你也试试看～" |
| 质疑/纠错 | 虚心接受 | "说得对，感谢指正！" |
| 广告/骚扰 | **不回复** | — |

### 回复原则

- 控制在 **≤100 字**
- 口语化、温暖、有人情味
- 不暴露 AI 身份
- 不在评论区放联系方式/外链（平台违规）
- 遇到恶意评论/投诉 → 标注在结果中上报主智能体
- 每轮互动不超过 **20 条**回复

## 社区规范（红线）

以下行为会导致笔记被限流或账号处罚：

- **禁止外链**：正文和评论中不放站外链接
- **禁止联系方式**：不直接展示微信号/手机号/邮箱
- **禁止刷量**：不在他人笔记下批量评论引流
- **禁止搬运**：内容必须原创，不照搬其他平台
- **禁止虚假宣传**：不夸大效果、不伪造使用体验
- **禁止敏感内容**：政治、色情、暴力、赌博
- **软广标注**：合作推广需标注"合作"/"赞助"
