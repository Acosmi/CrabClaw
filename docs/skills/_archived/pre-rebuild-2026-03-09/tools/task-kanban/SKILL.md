# Task Kanban

用于任务看板数据读取与状态巡检。

## 覆盖方法
- `tasks.list`

## 使用流程
1. 调用 `tasks.list` 获取任务集合。
   - 已知 `runId` 时优先传 `taskId` 精确查询；结果会带最近步骤、阶段和错误标记。
2. 按状态分组（待办/进行中/阻塞/完成）检查异常堆积。
3. 对阻塞任务回溯关联会话、日志和审批记录。

## 风险控制
- 任务名与来源必须可追踪，避免匿名任务长期堆积。

## 成功判定
- 阻塞任务可定位根因，状态分布合理。
