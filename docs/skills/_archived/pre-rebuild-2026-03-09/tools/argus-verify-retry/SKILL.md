# Argus 验证与回退

适用于动作之后确认是否生效，以及等待界面变化。

## 要点

- 点击、提交、切页后优先 `argus_watch_for_change`
- 只在变化不明显时再补局部读屏或截图
- 如果预期变化没出现，要先回报卡点，再决定是否重试

## 避免

- 每个动作后都重新整屏截图
- 未确认超时或失败原因就重复执行同一动作

## 故障树

- `argus_watch_for_change` 一直无变化：先区分是监视区域不对，还是前一步动作根本没生效
- 超时后还是不知道状态：说明需要补局部读屏，而不是立刻重试动作
- 变化很多但无法判断是否成功：说明验收信号定义太模糊，需补成功标志
- 进入“重试一次看看”循环：通常是没有先回报卡点

## 回滚步骤

- 先停止继续重试，把“期望变化是什么”说清
- 对变化区域做一次最小复查，优先局部读屏，其次必要截图
- 如果确认前一步没生效，回到对应动作技能修正，而不是直接再点一次
- 连续两轮仍不清楚时，先回报当前卡点和已观察到的状态

## 验收清单

- 已明确看到预期变化，或明确确认“没有变化”
- 重试次数受控，没有形成无限循环
- 复查使用的是最小观察手段，不是默认整屏截图
- 最终状态足以决定继续、回退或向主智能体汇报
