# 异步执行汇报技能

## 何时汇报

- 多阶段任务的阶段边界处
- 完成关键步骤、进入下一阶段时
- 被审批/构建/测试/外部资源阻塞时
- **不要**在短单步任务中汇报

## report_progress 参数

| 参数 | 必需 | 说明 |
|------|------|------|
| `summary` | 是 | ~300 字符以内的阶段描述 |
| `percent` | 否 | 1-100，仅在范围有效时使用 |
| `phase` | 否 | `analyzing` \| `implementing` \| `testing` 等 |

## 好的汇报示例

- "代码扫描完成，开始修改权限路径"
- "构建完成，正在运行回归测试"
- "审批已发送，等待确认"

## 关键边界

| 边界 | 说明 |
|------|------|
| 进度 ≠ 最终结果 | 汇报后必须仍然交付最终结果 |
| 投递可达性 | WebSocket/UI 实时可见；远程频道可能被节流 |
| OpenAI SSE | 不保证转发进度事件 |
| 文件产出 | 进度无法投递文件，需单独用 `send_media` |

## 常见错误

1. **高频汇报**（每条命令/每次截图都报）→ 日志流滥用
2. **无最终结果**（把进度当回复）→ 用户收不到结论
3. **假设全通道可见**（实际只有 UI 看得到）→ 远程用户无感知
