# 技能 3: 复核审计

> "LGTM" 不是审计。每一行代码都必须证明自己的存在价值。

## 适用场景

Crab Claw 包含安全边界代码（沙箱、权限审批、OAuth、网络隔离）。
粗略审查如果遗漏路径遍历、未检查的错误分支或竞态条件，后果严重。
此技能强制执行严格的逐行分析，并输出结构化审计报告。

## 规则

### 3.1 审计深度要求

**最低深度：对所有变更或新增代码逐行审计。**

以下敷衍回应**明确禁止**：

- "代码看起来没问题 / LGTM"
- "逻辑看起来正确"
- "没有明显问题"
- 只概述代码功能而不分析正确性

每次审计必须展示**主动推理**：
- 正常条件下每个代码路径做什么
- 错误/边界条件下每个代码路径做什么
- 代码做了哪些假设，这些假设是否成立

### 3.2 审计检查清单

每份审计报告必须覆盖所有适用类别：

#### 安全性
- [ ] **路径遍历**: 用户输入能否逃离预定目录？（检查 `..`、符号链接跟随、TOCTOU）
- [ ] **权限边界**: 隔离屏障是否正确应用？（沙箱策略、namespace、ACL、seccomp）
- [ ] **权限提升**: 受限进程能否获取不应有的能力？
- [ ] **输入验证**: 所有外部输入（CLI 参数、环境变量、IPC 消息、文件内容）是否在使用前验证？
- [ ] **注入风险**: SQL/命令/XSS 注入向量是否已消除？

#### 资源安全
- [ ] **错误路径清理**: 每个 `Err` 返回时，资源（FD、进程、临时文件、锁）是否正确释放？
- [ ] **并发安全**: 共享资源是否有保护？锁顺序是否一致？是否在持锁时 `.await`？
- [ ] **泄漏风险**: 是否有代码路径可能遗留僵尸进程、打开的 FD 或临时文件？
- [ ] **Goroutine/Task 泄漏**: 启动的 goroutine/tokio task 是否有取消/join 路径？

#### 正确性
- [ ] **边界情况**: 空输入、最大长度、Unicode、null 字节、并发访问
- [ ] **平台差异**: 代码在目标平台（macOS/Linux/Windows）上行为是否正确？
- [ ] **类型安全**: `as` 转换是否安全？Go 的类型断言是否有 ok 检查？
- [ ] **错误传播**: 错误是否带有足够上下文？是否有静默吞错误？

#### 交叉验证（跨语言适用）
- [ ] **Go ↔ Rust 一致性**: RPC 参数/返回值类型是否两端匹配？
- [ ] **前端 ↔ 后端一致性**: API 契约字段名、类型、可选性是否对齐？
- [ ] **依赖雷达**: 变更是否引入显式或隐式依赖？上下游是否已更新？

### 3.3 审计报告格式

存放于 `docs/claude/audit/`，命名规范：
`audit-YYYY-MM-DD-<组件>-<简述>.md`

```markdown
# 审计报告: <组件> — <简述>

## 范围
- 审查文件: [文件列表及行范围]
- 变更原因: [触发本次审计的描述]

## 发现

### [PASS | WARN | FAIL] <类别>: <简要标题>
- **位置**: `file.rs:LL-LL` 或 `file.go:LL-LL`
- **分析**: <关于正确性/错误的详细推理>
- **风险**: 无 | 低 | 中 | 高 | 严重
- **建议**: <如适用，具体修复方案>

（每个发现重复上述格式）

## 总结
- 发现总数: N (X 通过, Y 警告, Z 失败)
- 阻塞问题: [列表或"无"]
- 建议: [通过 | 有条件通过 | 拒绝]
```

### 3.4 审计触发条件

以下情况**必须**执行审计：

1. 将任何条目移至 `guidang/` 归档（SOP 工作流联锁）
2. 完成追踪文档的最终 checkbox
3. 引入或修改 `unsafe` 代码（Rust）
4. 修改权限审批、安全级别、沙箱隔离逻辑
5. 修改进程生命周期管理（spawn、kill、wait）
6. 修改 OAuth / 认证 / 凭证处理
7. 用户明确请求审查

### 3.5 归档红线

**绝对禁止在审计通过前归档。** 这是不可协商的铁律。

审计后操作流程：
1. 保存报告到 `docs/claude/audit/`
2. 更新原始追踪文档，标注审计报告路径
3. 如发现包含 FAIL 级条目：
   - 为所需修复创建新追踪项
   - 阻塞归档门控直到修复完成并复审
4. 所有发现 ≤ WARN 级 → 更新 verdict 为 Pass / Pass with Notes → 允许归档
