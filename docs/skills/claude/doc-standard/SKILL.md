# 技能 4: 文档规范

> 文档不是附属品，是工程产出的一等公民。

## 适用场景

`docs/claude/` 目录是 Crab Claw 项目的工程记忆。
不规范的文档会导致信息腐化、跨 session 丢失上下文、重复劳动。
此技能定义文档体系的铁律、目录结构和生命周期状态机。

## 目录结构

```
docs/claude/
├── renwu/          # 任务追踪 — 活跃任务分解与进度
├── tracking/       # 专项追踪 — 长期功能追踪文档
├── goujia/         # 架构与审计 — shenji-* 审计速记、arch-* 架构说明
├── guidang/        # 归档 — 已完成且审计通过的条目（只读）
├── deferred/       # 延迟项 — TODO、阻塞项、技术债
├── audit/          # 审计报告 — 技能 3（复核审计）的正式输出
└── archive/        # 历史归档 — 旧版材料存放
```

## 规则

### 4.1 目录用途铁律

| 目录 | 允许内容 | 禁止内容 |
|------|---------|---------|
| `renwu/` | 活跃任务追踪（带 checkbox） | 已完结任务、架构讨论 |
| `tracking/` | 跨 session 长期追踪 | 临时笔记、一次性说明 |
| `goujia/` | 架构文档 `arch-*`、审计速记 `shenji-*` | 任务追踪、延迟项 |
| `guidang/` | 已审计通过的完结文档 | 未审计的文档 |
| `deferred/` | 延迟项、技术债、未来计划 | 已完成的条目（应清理或归档） |
| `audit/` | 正式审计报告 | 非审计内容 |

### 4.2 文档生命周期状态机

```
 [创建] ──► [活跃] ──► [执行中] ──► [待审计] ──► [已归档]
   │           │          │            │            │
   renwu/    renwu/     renwu/      audit/ 生成   guidang/
   tracking/  tracking/  tracking/   报告后移动
```

状态转换规则：
- **创建 → 活跃**: 文档创建后立即进入活跃状态
- **活跃 → 执行中**: 开始勾选 checkbox 条目
- **执行中 → 待审计**: 所有 checkbox 完成，等待技能 3 审计
- **待审计 → 已归档**: 审计报告 verdict 为 Pass → 移至 `guidang/`
- **任何状态 → 延迟**: 发现阻塞项 → 记录到 `deferred/` 并在原文档标注

### 4.3 文档命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 审计报告 | `audit-YYYY-MM-DD-<组件>-<简述>.md` | `audit-2026-03-08-models-dual-track.md` |
| 追踪文档 | `tracking-<功能名>-YYYY-MM-DD.md` | `tracking-browser-retrofit-2026-03-07.md` |
| 架构文档 | `arch-<主题>.md` | `arch-cross-session-messaging.md` |
| 审计速记 | `shenji-NNN-<简述>.md` | `shenji-021-clearance-audit.md` |
| 延迟项 | `<功能名>.md` | `persistent-sandbox-worker.md` |
| 任务文档 | `<频道/模块>_<类型>_tracker.md` | `telegram_audit_tracker.md` |

### 4.4 文档内容规范

#### 必须包含
- **标题**: 一级标题说明文档主题
- **范围**: 明确覆盖的模块/文件/功能
- **状态标记**: 当前状态（活跃/执行中/待审计/已归档）
- **变更日期**: 最后更新日期

#### 禁止事项
- 禁止在 `guidang/` 中修改已归档文档（只读）
- 禁止只在对话中记录关键决策而不写入文档
- 禁止创建无法归类的散落文档（必须放入正确目录）
- 禁止重复创建已有文档（先搜索再创建）

### 4.5 SOP 六步流程 (A-F)

| 步骤 | 动作 | 输出 |
|------|------|------|
| A. 接收 | 理解任务范围，确认边界 | 任务描述（心理模型） |
| B. 分解 | 拆分为可执行的 checkbox 条目 | `renwu/` 追踪文档 |
| C. 执行 | 逐项实现，勾选 `[x]` | 代码变更 + 追踪更新 |
| D. 审计 | 触发技能 3（复核审计） | `audit/` 审计报告 |
| E. 修复 | 修复审计 FAIL/WARN 级发现 | 代码变更 + 复审 |
| F. 归档 | 审计通过后移至 `guidang/` | 完结文档迁移 |

### 4.6 延迟项管理

延迟项文件应包含：
- 来源：哪个任务/审计发现的
- 状态：活跃 / 已合并 / 已归档
- 优先级：阻塞 / 高 / 中 / 低
- 描述：具体内容和上下文
- 预计处理时机

定期清理（晴空行动）：
- 已完成的延迟项标记为 Archived
- 可合并的项目合并为一个
- 不再相关的项目删除并注明原因
